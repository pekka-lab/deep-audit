# Code AI Smell Patterns

Patterns in source code that reveal it was generated by AI. These should be cleaned up
to make the codebase feel intentional and human-crafted.

## 1. Leftover TODO/FIXME Comments

AI generators leave TODO comments for things they couldn't figure out or deferred.
Many never get resolved.

**Scan**: `grep -rn "TODO\|FIXME\|HACK\|XXX\|TEMP" --include="*.ts" --include="*.tsx"`

**Action**: For each one:
- If the thing it refers to is actually done: delete the comment
- If it's genuinely unfinished: log it in AUDIT.md as a known gap, then delete the comment
- Never leave TODOs in shipping code — they're either done or they're tracked issues

## 2. Over-Commenting Obvious Code

AI loves to explain every line, even trivial ones.

**Examples of comments to remove**:
```typescript
// Import React
import React from 'react';

// Define the user interface
interface User {
  // The user's name
  name: string;
  // The user's email
  email: string;
}

// Create the component
export function UserCard({ user }: { user: User }) {
  // Return the JSX
  return <div>{user.name}</div>;
}
```

**Action**: Remove comments that say what the code already says. Keep comments that explain WHY
something non-obvious is done a certain way.

## 3. console.log Left in Production Code

AI often adds console.log for debugging and forgets to remove them.

**Scan**: `grep -rn "console\.log\|console\.warn\|console\.error\|console\.debug" --include="*.ts" --include="*.tsx" --include="*.js"`

**Action**:
- Remove all `console.log` and `console.debug`
- Keep `console.error` only if it's in a catch block and the error is genuinely unexpected
- Keep `console.warn` only if it warns about a real deprecation or misconfiguration

## 4. Dead Code

Functions, components, or utilities that are defined but never imported or used anywhere.

**How to find**:
- For exported functions: grep for the function name across the codebase
- For components: check if the component file is imported anywhere
- TypeScript compiler with `noUnusedLocals` and `noUnusedParameters` catches local dead code

**Action**: Delete dead code entirely. Don't comment it out. Git has history.

## 5. Unnecessary Wrapper Functions

AI creates abstractions "just in case" that add a layer without adding value.

**Pattern**:
```typescript
// A function that does nothing but call another function
function fetchUserData(userId: string) {
  return supabase.from('users').select('*').eq('id', userId).single();
}

// A wrapper component that just passes props through
function UserNameDisplay({ name }: { name: string }) {
  return <span>{name}</span>;
}
```

**Action**: Inline the call at the usage site. Remove the wrapper.

Exception: Keep wrappers that genuinely add error handling, logging, or transform the response.

## 6. Inconsistent Naming

AI doesn't always remember what convention it used earlier in the codebase.

**Common inconsistencies**:
- Files: some `kebab-case.ts`, some `camelCase.ts`, some `PascalCase.ts`
- Variables: mixing `camelCase` and `snake_case`
- Components: some use `FC<Props>`, some use plain function with typed args
- Hooks: some in `hooks/`, some in `lib/`, some co-located

**Action**: Pick the dominant convention in the codebase and normalize to it.
Don't change everything — just fix the outliers.

## 7. Generic Variable Names at Module Scope

Local `data` in a small function is fine. Module-level `data` or `result` is not.

**Pattern**:
```typescript
// Bad: what data? what result?
const data = await fetchData();
const result = processData(data);

// Good: names tell you what they contain
const userProfile = await fetchUserProfile();
const formattedProfile = formatForDisplay(userProfile);
```

**Action**: Rename module-scope generics to describe what they actually contain.

## 8. Placeholder Error Messages

AI writes generic error handling that gives users no useful information.

**Pattern**:
```typescript
catch (error) {
  toast.error("Something went wrong");
  // or
  toast.error("An error occurred. Please try again.");
  // or
  console.error("Error:", error);
  return { error: "Failed to process request" };
}
```

**Action**: Make error messages specific and actionable:
- "Could not save your changes. Check your internet connection and try again."
- "This email address is already registered. Try signing in instead."
- "File too large. Maximum size is 10MB."

## 9. Excessive Type Assertions

AI uses `as` casts and `!` non-null assertions to make TypeScript happy without fixing the underlying type issue.

**Scan**: `grep -rn " as \| as any\|!\." --include="*.ts" --include="*.tsx"`

**Action**: Replace `as` casts with proper type narrowing (type guards, optional chaining).
Remove `as any` entirely — fix the actual type mismatch.

## 10. Copy-Paste Patterns

AI sometimes generates similar blocks of code in multiple places instead of extracting a shared function.

**How to spot**: Look for files with very similar structure, or blocks of code that repeat
with minor variations (different field names, same shape).

**Action**: Only extract if there are 3+ repetitions. Two is fine. Don't create a premature abstraction.
